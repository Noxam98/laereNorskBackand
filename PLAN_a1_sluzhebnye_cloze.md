# План: служебные слова A1 + cloze

> Живой документ. Правим вместе. У каждой развилки есть **Дефолт** (моё предложение)
> и строка **РЕШЕНИЕ:** — заполняем по мере согласования. Статусы: ⬜ обсуждаем · ✅ решено · 🔄 в работе · ✔️ сделано.

## Цель
Служебные слова (`og, i, å, på, til, ikke…`) должны учиться осмысленно:
- вводиться **после** базы контентных слов (есть из чего строить фразу);
- через **простые карточки с примерами** и **cloze** («вставь пропуск»), собранный из уже знакомых слов;
- без заумных глоссов («å → инфинитивная частица») и без странной TTS голой частицы.

## Контекст (как сейчас)
- Рампа (`REQUIRED_CELLS`): `choice_no2int → choice_int2no → build_int2no → input_int2no → ВЫУЧЕНО`; до неё карточка-интро (study) при `attempts==0`.
- Ошибка → откат на 1 ступень назад (вариант B), карточку не трогает.
- «Умная очередь»: только что показанное слово не лидирует следующую сессию (кулдаун 12 мин).
- Дистракторы: ANN-соседи; синонимы по смыслу исключаются.
- Модель: `gemini-3.1-flash-lite` (фолбэк `gemini-3.5-flash`). Описания есть, примеров-предложений нет.
- Найдено: `å` pos=verb, ru=«инфинитивная частица» (jargon). Служебных по POS ~30–40.

## Фазы
- **Ф0. Контент** — починить глоссы + простой пример для служебных.
- **Ф1. Класс «служебное» + гейт ввода** — вводить после N контентных. *(готовит почву, ценно само)*
- **Ф2. Карточка служебного** — простой пример, TTS примера.
- **Ф3. Урезанная рампа** — для служебных вместо build/input → cloze.

## Развилки и решения

### 1. Что считается «служебным» (POS-класс) ✅
**Дефолт:** `preposisjon, konjunksjon, pronomen, determinativ, interjeksjon` + частица `å`.
Спорные `ikke`/`inn`/прочие наречия — **по ручному вайтлисту** (наречий много контентных, скопом нельзя).
**РЕШЕНИЕ:** Вариант **B** — ядро `konjunksjon + preposisjon + pronomen + determinativ + å`
**+ ручной вайтлист функциональных наречий** (`ikke, inn, ut, opp, ned, her, der, nå, da, også`…).
Междометия и числительные — обычные слова (не служебные).

### 2. Гейт ввода: порог N контентных ✅
**Дефолт:** N = **12**. «База гейта» = `learning + mastered` (мягче). «Костяк cloze» = строго `mastered`.
**РЕШЕНИЕ:** База гейта = **`learning + mastered`**; **N = 20**. Костяк cloze — строго `mastered`.
Отдельного исключения для `jeg/og` не нужно (cloze строим из контентных: «kaffe ___ te»).

### 3. Карточка служебного: пример откуда ✅
**Дефолт:** карточка-интро — **глобальный** простой канонический пример (дёшево, один раз): «og — и · _kaffe og te_». Cloze — **персональный**, из выученных. (Интро нужно ДО того, как есть база для персонального.)
**РЕШЕНИЕ:** Вариант **A** — на карточке глобальный канонический пример (один раз на слово);
персонализация — только в cloze.

### 4. Чем чинить заумные глоссы (~30–40 слов) ✅
**Дефолт:** разовый **LLM-батч** (через фон-воркер приложения, не мой ssh). Альтернатива — вручную.
**РЕШЕНИЕ:** Вариант **A** (разово + ревью), но генерит **сам Claude локально** (не Gemini) —
надёжнее, без квоты/ssh. Claude выдаёт {упрощённый перевод 5 яз. + канонический A1-пример} по ~40
служебным → я показываю списком → после «ок» записываю в пул. Сам перевод УПРОЩАЕМ (заумь убираем),
нюанс — в пример/описание.

### 5. Формат cloze ✅
**Дефолт:** **выбор из вариантов** (дистракторы = другие служебные той же POS). Печатать «og» по буквам бессмысленно.
**РЕШЕНИЕ:** Вариант **A** — выбор из вариантов (переиспользуем ChoiceQuestion), дистракторы по POS (#8).

### 6. Урезанная рампа для служебных (клетки) ✅
**Дефолт:** `card → choice_no2int → choice_int2no → cloze` (без build/input; cloze = «продукция»). «Выучено» — после cloze.
**РЕШЕНИЕ:** Рампа служебного = **`карточка-интро → cloze ×3 (три РАЗНЫХ предложения)` → выучено**.
Без choice/build/input. Каждый из 3 cloze — отдельное предложение/контекст; «выучено» после 3 успешных.
Ошибка — откат на 1 шаг (как везде, см. ramp-demotion B).
Открытые под-вопросы: как считать «успешно» (первая попытка) и генерить 3 предложения сразу или по одному → см. #7 и #10.

### 7. Cloze: модель и надёжность ✅
**Дефолт:** генерит **фон-воркер приложения** на `flash-lite`; конвейер «сгенерь → бесплатная проверка `использованные леммы ⊆ известные` (модель сама перечисляет леммы) → verify однозначности → при сбое перегенерь / эскалация на `3.5-flash`». Кэш **per (user, word)**, lazy при выходе на ступень cloze.
**РЕШЕНИЕ:** Вариант **B** — генерим cloze на **`gemini-3.5-flash`** (проверено вживую: даёт чистые A1-пропуски).
⚠️ **3.5-flash — reasoning-модель:** нужен щедрый `max_tokens` (≥256–512), иначе `content` пустой (бюджет уходит на «размышление»). Учесть в вызове.
Конвейер: 3 предложения батчем (#10в) из mastered-словаря (#2) → проверка `леммы ⊆ известные` → verify однозначности (принимаем перечисленные доп. верные) → кэш per (user, word). flash-lite — фолбэк.

### 8. Дистракторы для choice/cloze служебных ✅
**Дефолт:** **по POS-классу** (og↔i↔til↔på), не ANN.
**РЕШЕНИЕ:** Вариант **A** — дистракторы по POS-классу (исключая синонимы по смыслу);
для cloze доп. фильтр: дистрактор не должен сам подходить в пропуск (= verify однозначности из #7).

### 9. Буквы алфавита ✅
**Дефолт:** отдельной обработки **не делаем** — «чистых букв» в пуле нет (`å`/`i` лежат как слова).
**РЕШЕНИЕ:** Отдельной обработки нет. Проверено: однобуквенных всего 2 — `å` (частица) и `i` (предлог),
оба настоящие слова и так попадают в класс служебных.

### 10. Состав сессии: cloze-слова ✅
Требование: **в сессии всегда несколько cloze-слов, когда есть из чего** (не вся сессия из cloze, но и не пусто).
**РЕШЕНИЕ:**
- (а) **до 3 cloze-слов на сессию** (если назрели; меньше/ноль — если их мало);
- (б) «успешно» = **первая попытка верна**; 3 разных предложения (`cloze_1/2/3`); промах → откат на шаг; «выучено» после 3;
- (в) **генерим 3 предложения сразу батчем** при выходе на стадию cloze, кэшируем.

## Открытые риски (держим в уме)
- Однозначность пропуска для предлогов («går ___ skolen» — på/til) → принимаем несколько верных ответов.
- Флексии vs леммы в проверке костяка → модель перечисляет использованные леммы.
- Инвалидация cloze-кэша при росте словаря → редкая регенерация, не критично.
- Ф4 трогает фронт (новый тип игры в `useGameLoop`/`LearningSession`) — впишется, но самый большой кусок.

## Итоговый дизайн (все развилки решены)
- **Класс «служебное»** = konjunksjon + preposisjon + pronomen + determinativ + `å` + ручной вайтлист функц. наречий (`ikke, inn, ut, opp, ned, her, der, nå, da, også`…). Междометия/числительные — обычные.
- **Гейт ввода:** служебное вводится новым только при ≥ **20** контентных в `learning+mastered`. Костяк cloze — строго `mastered`.
- **Рампа служебного:** `карточка-интро → cloze ×3 (3 разных предложения)` → выучено. Без choice/build/input. Промах → откат на 1 шаг.
- **Карточка:** глобальный канонический пример (один на слово).
- **Глоссы+канонический пример** (~40 слов): пишет **Claude вручную** (упрощаем перевод), ревью → запись.
- **Cloze-формат:** выбор из вариантов; дистракторы по POS-классу (не подходящие в пропуск); принимаем неск. верных.
- **Cloze-генерация:** `gemini-3.5-flash` (reasoning! `max_tokens`≥256–512), 3 предложения батчем из mastered-словаря; проверка `леммы⊆известные` (модель перечисляет леммы) + verify однозначности; кэш per (user,word), lazy.
- **Сессия:** до 3 cloze-слов, когда назрели.
